## Tag Comparator

## Objective

Implement the **Tag Comparator** module to determine whether a CPU memory request results in a **Cache Hit** or a **Cache Miss**. The comparator checks the requested tag generated by the Address Decoder against the stored tag from the Tag RAM while considering the cache line's **Valid Bit**.

## Completed

* Designed a parameterized Tag Comparator in SystemVerilog.
* Compared the requested tag with the stored cache tag.
* Implemented cache hit detection.
* Implemented cache miss detection.
* Considered the Valid Bit during comparison.
* Developed a standalone SystemVerilog testbench.
* Verified all possible comparison scenarios.

## Architecture

```text
                Requested Tag
                      │
                      ▼
              +----------------+
              | Tag Comparator |
              +----------------+
               ▲            ▲
               │            │
          Stored Tag    Valid Bit
               │            │
               └──────┬─────┘
                      │
                      ▼
                Cache Hit / Miss
```

## Module Interface

### Inputs

| Signal       | Description                          |
| ------------ | ------------------------------------ |
| `req_tag`    | Tag extracted from the CPU address   |
| `stored_tag` | Tag stored in the Tag RAM            |
| `valid`      | Valid bit of the selected cache line |

### Outputs

| Signal | Description                                                  |
| ------ | ------------------------------------------------------------ |
| `hit`  | Asserted when tags match and the cache line is valid         |
| `miss` | Asserted when tags do not match or the cache line is invalid |

## Cache Hit Logic

A cache hit occurs only when:

```text
Valid Bit = 1
AND
Requested Tag == Stored Tag
```

Otherwise, the access is considered a **Cache Miss**.

## Features

* Parameterized tag width
* Pure combinational logic
* Independent of clock
* Low hardware complexity
* Modular design for easy integration
* Suitable for direct-mapped cache architecture

## Test Cases

| Test Case                             | Expected Result | Status |
| ------------------------------------- | --------------- | ------ |
| Matching tags with valid bit set      | Cache Hit       | ✓      |
| Different tags with valid bit set     | Cache Miss      | ✓      |
| Matching tags with valid bit cleared  | Cache Miss      | ✓      |
| Different tags with valid bit cleared | Cache Miss      | ✓      |
